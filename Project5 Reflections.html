Implementation Section :


a) The Model :

The overal model of this project is such that the Simulator sends the following parameters:

1) a set of waypoints in global coordinates. This set consistutes the trajectory for the car to follow.

2) Location of the vehicle in global/map coordinates

3) Orientation of the vehicles in radians

4) Velocity of the vehicle

5) Steering angle of the vehicle

6) the throttle value where -1 is full brake – 0 neutral --- +1 full trottle

And the task is to provide the proper input, based on the parameters above and latency estimation into the MPC engine along with constraints and cost weights for it to generate the optimum steering angle and throttle value which then is sent back to the Simulator and drive the vehicle around the track safely.


In addition, for visual feedback purposes, the waypoints/ provided trajectory and the predicted trajectory points, in vehicle coordinates, is provided back to the Simulator to display.


b) Timestep Length and Elapsed Time Duration:

I chose a timestep length of 10 with duration of 100ms.





c) Poynomial fitting :

Given that the the vehicle was to drive around a track with fairly tight turns, I chose to use a third order polynomial to fit the trajectory. This was done after the provided trajectory points were transformed to vehicle coordinates.

















d) Model Predictive Control with Latency:

In order to have the MPC model to produce optimum values, the input state need to account for the delay (latency) due to getting values from the simulator and the time we feed into the MPC engine. This latency was simulated with value of 100ms. As such, all time-dependent state values needed to be adjusted. For example, the forward position of the car needed to be adjusted by v_in * 100ms i.e. the distance the car would travel in 100ms given the car’s velocity

The vehicle velocity got adjusted by the throttle / acceleration value v = v_in + throttle_in * 100ms

Similarly the cross-track-error (cte) got adjusted by the value that takes into account change in orientation angle of the vehicle orientation.

With these properly adjusted input state variables we ensure that the MPC model has accurate values.
