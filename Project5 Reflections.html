Implementation Section :


a) The Model :

The overal model of this project is such that the Simulator sends the following parameters:

1) a set of waypoints in global coordinates. This set consistutes the trajectory for the car to follow.

2) Location of the vehicle in global/map coordinates

3) Orientation of the vehicles in radians

4) Velocity of the vehicle

5) Steering angle of the vehicle

6) the throttle value where -1 is full brake – 0 neutral --- +1 full trottle

And the task is to provide the proper input, based on the parameters above and latency estimation into the MPC engine along with constraints and cost weights for it to generate the optimum steering angle and throttle value which then is sent back to the Simulator and drive the vehicle around the track safely.


In addition, for visual feedback purposes, the waypoints/ provided trajectory and the predicted trajectory points, in vehicle coordinates, is provided back to the Simulator to display.



b) Timestep Length and Elapsed Time Duration:


I chose a timestep length of N=10 with duration dt= 100ms. The decision that drives towrds determining N and dt is their product T (T = N *dt) By choosing T, we choose the duration of the Trajectory, how far out in time are we trying to predict. proper controller /actuator values. Then we determine actual values of N and dt.

Higher values of implies lower values of dt which implies longer calculations in shorter amounts of time (dt)… This is important given that the MPC model throws away all but the first calculation after a single step in order to recalculate .

Smaller number of steps implies longer time increments dt, this will result in introducing inaccuracy per each step as we will re-calculate for the next step after a longer period of time.

d) Model Predictive Control with Latency:


In order to have the MPC model to produce optimum values, the input state need to account for the delay (latency) due to getting values from the simulator and the time we feed into the MPC engine. This latency was simulated with value of 100ms. As such, all time-dependent state values needed to be adjusted:

The forward position x of the car is adjusted predicted_x = x0 + v_in * 100ms where x0 = 0 because the current location of the car is (0,0) I. This would account for the distance the car would travel in 100ms given the car’s velocity .

The vehicle velocity v got adjusted by the throttle / acceleration value predicted_v = v_in + throttle_in * 100ms.

Similarly the cross-track-error (cte) got adjusted by the value that takes into account change in orientation angle of the vehicle orientation. predicted_cte = cte + v_in * sin(epsi) * 100ms.

Finally, predicted_epsi = epsi + (v_in/Lf * steering_angle * 100msec)

With these properly adjusted input state variables we ensure that the MPC model has accurate values.
